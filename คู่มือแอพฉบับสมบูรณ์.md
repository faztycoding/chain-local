# คู่มือแอพฉบับสมบูรณ์ — Chain Counter & AI Defect Inspection

เอกสารนี้อธิบายทุกอย่างของระบบ ตั้งแต่วิธีติดตั้ง, หน้าเว็บทั้ง 4 หน้า, API, ฐานข้อมูล, ฟีเจอร์จำลอง AI, ไปจนถึงวิธีเอาไปใช้จริงที่โรงงาน

---

# 1) ระบบนี้คืออะไร

เว็บแอปสำหรับงานตรวจสอบโซ่ มีหน้าที่หลัก 2 อย่าง:

- **นับข้อโซ่**
- **ตรวจจับตำหนิจาก AI**

Flow ของระบบ:

```
สร้างคำสั่ง → AI ตรวจโซ่ → ผลอัปเดตเรียลไทม์ → สรุปเป็นรายงาน
```

ตัวละครหลัก 6 ตัว:

| ตัวละคร | หน้าที่ |
|---------|---------|
| **หน้า Input** | สร้างคำสั่งตรวจ |
| **หน้า Output** | แสดงผลเรียลไทม์ + จำลอง AI |
| **หน้า Stats** | สรุปรายงาน + กราฟ |
| **Server** | รับ-ส่งข้อมูลทุกอย่าง |
| **Database** | เก็บข้อมูลทั้งหมด (SQLite) |
| **AI (YOLOv8)** | ตรวจภาพแล้วส่งผลกลับ |

# 2) วิธีติดตั้งและเริ่มใช้งาน

## สิ่งที่ต้องมี
- **Node.js** (เวอร์ชัน 18 ขึ้นไป)
- **ไม่ต้องต่อ internet** หลังจากติดตั้งครั้งแรก

cd chain-local

# 2. ติดตั้ง dependencies (ทำครั้งเดียว)
npm install

# 3. เปิด server
npm start

| หน้า | URL |
|------|-----|

| หน้าจอควบคุม | http://localhost:3000/page2_output.html |

# 3) หน้าเว็บทั้ง 4 หน้า

## หน้า 1: สั่งงานตรวจสอบ (Input)

**ไฟล์:** `public/page1_input.html` + `public/js/input.js`

สร้างคำสั่งตรวจใหม่ โดยกรอก:
- โหมดการตรวจ (นับ / ตรวจตำหนิ / ทำทั้งสอง)
- ขนาดโซ่ (6mm, 8mm, 10mm, ...)
- สีโซ่ (เงิน, ทอง, ดำ, ...)
- รายละเอียดสินค้า (ไม่บังคับ)

เมื่อกดส่ง → ข้อมูลบันทึกลง DB + หน้า Output อัปเดตทันที

info chip 3 อันด้านล่างฟอร์ม ("บันทึกลงฐานข้อมูลทันที", "ส่งต่อหน้า Output อัตโนมัติ", "พร้อมเชื่อม AI ต่อ") เป็นแค่ป้ายบอกว่าระบบทำอะไรให้ ไม่ใช่ปุ่มกด

---

## หน้า 2: หน้าจอควบคุม (Output)

**ไฟล์:** `public/page2_output.html` + `public/js/output.js`

แสดงผลเรียลไทม์ + ควบคุมสถานะ + จำลอง AI

**ส่วนประกอบ:**

- **Dropdown เลือกคำสั่ง** — เลือกดูคำสั่งเก่า ๆ ได้ ไม่แสดงแค่อันล่าสุด
- **การ์ดข้อมูลคำสั่ง** — โหมด, ขนาด, สี, จำนวนข้อรวม, ตำหนิรวม
- **ผลการตรวจจาก AI** — ภาพ, ประเภทตำหนิ, ความมั่นใจ
- **Log เหตุการณ์** — บันทึกทุก event ที่เกิดขึ้นตามลำดับเวลา

**ปุ่มควบคุมใน Sidebar:**

| ปุ่ม | สิ่งที่ทำ |
|------|----------|
| **เริ่มทำงาน** | เปลี่ยนสถานะ → running (เขียว) |
| **หยุดทำงาน** | เปลี่ยนสถานะ → stopped (แดง) |
| **หยุดฉุกเฉิน** | เปลี่ยนสถานะ → emergency (แดง) |
| **🤖 จำลองผล AI** | สุ่มผลตรวจจำลองส่งเข้าระบบ (สีม่วง) |

ปุ่มเริ่ม/หยุด/ฉุกเฉิน เปลี่ยนสถานะใน DB + อัปเดตหน้าจอจริง แต่ยังไม่ได้ต่อเครื่องจักรจริง (รอเชื่อมภายหลัง)

## หน้า 3: สถิติการผลิต (Stats)

**ไฟล์:** `public/page3_stats.html` + `public/js/stats.js`

แดชบอร์ดรายงานผลการตรวจสอบ มี 6 แท็บ:

| แท็บ | เนื้อหา |
|------|---------|
| ภาพรวม | การ์ดสรุป 5 ตัว + กราฟ 4 อัน |
| กราฟควบคุม (p-chart) | ดูว่ากระบวนการอยู่ในขีดควบคุมไหม |
| รายงานประจำวัน | สรุปรายวัน |
| รายงานรายสัปดาห์ | สรุปรายสัปดาห์ |
| รายงานรายเดือน | สรุปรายเดือน |
| ประวัติการตรวจสอบ | ดูผลตรวจทุกรายการ |

การ์ดสรุป 5 ตัว: จำนวนคำสั่งทั้งหมด, จำนวนการตรวจ, จำนวนตำหนิ, อัตราของเสีย (%), ความมั่นใจเฉลี่ย (%)

เมื่อยังไม่มีข้อมูล → แสดง empty state + ลิงก์ไปหน้า Input

# 4) ปุ่ม จำลองผล AI

ฟีเจอร์นี้มีไว้ **ทดสอบ/เดโมได้โดยไม่ต้องมี YOLOv8 จริง**

## วิธีใช้:
1. สร้างคำสั่งจากหน้า Input ก่อน
2. ไปหน้า Output
3. กดปุ่ม **🤖 จำลองผล AI** (สีม่วง ใน sidebar)
4. กดได้หลายครั้ง ผลจะสุ่มต่างกันทุกครั้ง

## ผลที่สุ่มได้:

| ผลลัพธ์ | ภาพ | โอกาส |
|---------|-----|-------|
| ผ่าน (ไม่พบตำหนิ) | กรอบเขียว ✓ | ~43% |
| รอยขีดข่วน (scratch) | กรอบแดง | ~14% |
| รอยร้าว (crack) | กรอบแดง + เส้นร้าว | ~14% |
| สนิม (rust) | กรอบส้ม + วงสนิม | ~14% |
| รูปทรงผิดปกติ (deformation) | กรอบแดง | ~14% |

เมื่อกด → ระบบยิง POST /api/detect เหมือน AI จริง → หน้าจออัปเดตทันที → หน้า Stats ก็มีข้อมูลเพิ่ม

# 5) Flow ทดสอบครบวงจร

ทำตามนี้จะเห็นระบบทำงานครบทุกส่วน:

```
1. หน้า Input    → สร้างคำสั่งตรวจ
2. หน้า Output   → เห็นคำสั่งอัปเดตทันที
3. กด "เริ่มทำงาน" → สถานะเป็นเขียว
4. กด "จำลองผล AI" หลายครั้ง → เห็นภาพ + ผลเปลี่ยนทุกครั้ง
5. หน้า Stats    → กราฟ + ตารางมีข้อมูลให้ดู
6. ดูแท็บต่าง ๆ  → p-chart, รายวัน, รายสัปดาห์, ประวัติ
```

# 7) API ทั้งหมด

## คำสั่งตรวจ (Orders)

| Method | Endpoint | หน้าที่ |
|--------|----------|---------|
| `POST` | `/api/orders` | สร้างคำสั่งใหม่ |
| `GET` | `/api/orders` | ดูคำสั่งทั้งหมด |
| `GET` | `/api/orders/:id` | ดูคำสั่งตาม id |
| `PATCH` | `/api/orders/:id/status` | เปลี่ยนสถานะ |

## AI Detection

| Method | Endpoint | หน้าที่ |
|--------|----------|---------|
| `POST` | `/api/detect` | รับผลตรวจจาก AI |

ตัวอย่าง JSON ที่ AI ต้องส่ง:

```json
{
  "order_id": 1,
  "chain_count": 12,
  "defect_type": "scratch",
  "defect_detail": "รอยขีดข่วนที่ข้อโซ่ที่ 3",
  "confidence": 0.92,
  "image_path": "/images/result_001.jpg"
}
```

defect_type ที่รองรับ: `none`, `scratch`, `crack`, `rust`, `deformation`

## หน้า Output

| Method | Endpoint | หน้าที่ |
|--------|----------|---------|
| `GET` | `/api/output/current` | คำสั่ง + ผลตรวจล่าสุด |
| `GET` | `/api/output/:id` | คำสั่ง + ผลตรวจตาม id |

## สถิติ

| Method | Endpoint | หน้าที่ |
|--------|----------|---------|
| `GET` | `/api/stats/export` | ข้อมูลผลตรวจทั้งหมด |
| `GET` | `/api/stats/daily` | สรุปรายวัน |
| `GET` | `/api/stats/weekly` | สรุปรายสัปดาห์ |
| `GET` | `/api/stats/monthly` | สรุปรายเดือน |
| `GET` | `/api/stats/pchart` | ข้อมูล p-chart |
| `GET` | `/api/stats/defect-by-color` | ตำหนิแยกตามสี |
| `GET` | `/api/history` | ประวัติทั้งหมด |

# 8) ฐานข้อมูล

ใช้ **SQLite** — เก็บเป็นไฟล์ `chain_data.db` ไม่ต้องติดตั้งฐานข้อมูลแยก

## ตาราง `orders`

| คอลัมน์ | ความหมาย |
|---------|----------|
| `id` | รหัสคำสั่ง (สร้างอัตโนมัติ) |
| `mode` | โหมดการตรวจ |
| `chain_size` | ขนาดโซ่ |
| `chain_color` | สีโซ่ |
| `product_attribution` | รายละเอียดสินค้า |
| `status` | สถานะ (pending/running/stopped/emergency) |
| `total_chain_count` | จำนวนข้อโซ่รวม (สะสม) |
| `total_defect_count` | จำนวนตำหนิรวม (สะสม) |
| `created_at` | เวลาสร้าง |
| `updated_at` | เวลาอัปเดตล่าสุด |

## ตาราง `inspection_results`

| คอลัมน์ | ความหมาย |
|---------|----------|
| `id` | รหัสผลตรวจ |
| `order_id` | เชื่อมกับคำสั่งไหน (สำคัญที่สุด!) |
| `chain_count` | จำนวนข้อที่ตรวจพบรอบนี้ |
| `defect_type` | ประเภทตำหนิ |
| `defect_detail` | รายละเอียดตำหนิ |
| `confidence` | ค่าความมั่นใจ |
| `image_path` | path รูปผลตรวจ |
| `timestamp` | เวลาตรวจ |

## ทำไมต้องมี `order_id`

เป็น **กุญแจเชื่อมข้อมูลหลัก**: 1 คำสั่ง → มีผลตรวจได้หลายครั้ง ถ้าไม่มี `order_id` ระบบจะไม่รู้ว่าผลตรวจเป็นของงานไหน

## View สำหรับรายงาน

| View | ใช้ทำอะไร |
|------|----------|
| `power_bi_inspection_summary` | รวมข้อมูลสำหรับ export ไป Power BI |
| `daily_qa_summary` | สรุปรายวัน (จำนวนตรวจ, ผ่าน, ตำหนิ, defect rate) |
| `pchart_data` | ข้อมูลสำหรับ p-chart ดูขีดควบคุมคุณภาพ |

# 9) Real-time (Socket.io)

ใช้ Socket.io เป็น "สายตรง" ระหว่าง server กับ browser — ข้อมูลอัปเดตทันทีโดยไม่ต้อง refresh

| Event | เกิดเมื่อ | ผลลัพธ์ |
|-------|----------|---------|
| `new_order` | สร้างคำสั่งใหม่ | หน้า Output แสดงคำสั่งใหม่ทันที |
| `detection_result` | AI ส่งผลตรวจมา | หน้า Output แสดงผล + ภาพ + Log ทันที |
| `order_status_changed` | กดเริ่ม/หยุด/ฉุกเฉิน | สถานะอัปเดตทันที |

# 10) การเอาไปใช้จริงที่โรงงาน

## ระบบนี้เป็น localhost ทั้งหมด ไม่มีอะไรขึ้น cloud

| ส่วน | ที่เก็บ |
|------|--------|
| Web Server | localhost:3000 (Node.js) |
| ฐานข้อมูล | ไฟล์ chain_data.db ในโฟลเดอร์ |

| รูปภาพ | โฟลเดอร์ public/images/ |

## วิธีเอาไปใช้ที่เครื่องโรงงาน:
1. ก็อปทั้งโฟลเดอร์ไปวาง
2. เครื่องนั้นต้องมี Node.js ติดตั้งไว้
3. เปิด Terminal → `npm install` → `npm start`
4. เปิด Browser → `localhost:3000` → ใช้งานได้เลย

## สิ่งที่ฝั่งเว็บทำเสร็จแล้ว:
- สร้างคำสั่งตรวจสอบ
- แสดงผลเรียลไทม์
- เก็บข้อมูลลง DB
- สถิติ + กราฟ + p-chart
- API พร้อมรับผลจาก AI
- ปุ่มจำลอง AI สำหรับเดโม
- Dropdown เลือกดูคำสั่งเก่า

## สิ่งที่ต้องรอเพื่อนฝั่ง AI เอามาต่อ:
- โมเดล YOLOv8 จริง
- กล้องถ่ายโซ่จริง
- Logic ตรวจจับจริง
- การส่งรูปจริง

ฝั่ง AI แค่เขียน Python ยิง `POST /api/detect` เข้ามาเป็น JSON ก็เชื่อมต่อกันได้เลย

# 11) ตัวอย่างโค้ดฝั่ง AI (Python)

```python
import requests

# ส่งผลตรวจเข้าระบบ
requests.post("http://localhost:3000/api/detect", json={
    "order_id": 1,
    "chain_count": 12,
    "defect_type": "scratch",
    "defect_detail": "รอยขีดข่วนที่ข้อโซ่ที่ 3",
    "confidence": 0.92,
    "image_path": "/images/result_001.jpg"
})
```

แค่นี้ผลจะขึ้นหน้า Output ทันที ไม่ต้องแก้โค้ดฝั่งเว็บอะไรเพิ่ม

# 12) สิ่งที่ต้องคุยกับเพื่อนฝั่ง AI

| หัวข้อ | รายละเอียด |
|--------|-----------|
| **AI จะรู้ order_id ยังไง** | อ่านจาก API `GET /api/orders` หรือกำหนด flow กลาง |
| **chain_count นับต่ออะไร** | ต่อเฟรม / ต่อภาพ / ต่อชิ้นงาน — ต้องตกลงให้ชัด |
| **defect_type ใช้คำอะไร** | none, scratch, crack, rust, deformation |

| **ส่งบ่อยแค่ไหน** | ทุกเฟรม / ทุกชิ้นงาน / ทุก N วินาที |

# 13) สรุป

ถ้าจะจำแบบง่ายที่สุด:

- **หน้า Input** = สร้างงาน
- **หน้า Output** = ดูผลเรียลไทม์ + จำลอง AI ได้
- **หน้า Stats** = สรุปรายงาน
- **server.js** = คนกลางรับ-ส่งข้อมูล
